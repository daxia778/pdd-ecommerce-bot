# 拼多多开放平台 API 授权及应用接入指南摘要

> 根据 2026-03 提供的 PDD 开放平台后台文档整理。

## 1. 授权核心原理
- **鉴权指标**：所有需要用户授权的接口（无论读写），均使用 `access_token` 作为唯一鉴权指标。
- **协议**：采用国际通用的 OAuth 2.0 标准协议。

### 授权流程图
`Client (客户端) -> HTTP 调用 -> 生成 sign 及封装 HTTP 请求 -> 公网 -> POP (拼多多服务端)`

## 2. 授权方式与入口
根据**账号类型**和**使用端**的不同，授权地址有所区分：
| 账号类型 | 授权形式 | 授权页链接 |
|---|---|---|
| 拼多多店铺 | WEB 端 | `https://fuwu.pinduoduo.com/service-market/auth` |
| 拼多多店铺 | H5 移动端 | `https://mai.pinduoduo.com/h5-login.html` |
| 多多进宝推手 | WEB 端 | `https://jinbao.pinduoduo.com/open.html` |
| 快团团团长 | WEB 端 | `https://oauth.pinduoduo.com/authorize/ktt` |
| 电子面单用户 | WEB 端 | `https://wb.pinduoduo.com/logistics/auth` |

## 3. 授权 URL 组装与参数
需要引导用户访问组装好的授权 URL 以获取 `code`，标准格式如下：
`https://[授权页链接]?response_type=code&client_id=[应用client_id]&redirect_uri={client_id对应的回调地址}`

**核心参数：**
*   `client_id` (必选): 应用详情中查看。
*   `response_type` (必选): 固定值为 `code`。
*   `redirect_uri` (必选): 应用的回调地址 (必须进行 `encodeURIComponent` 编码)。用户授权后，code 会回调到该地址。
*   `state` (可选): 自定义参数，原值带回。
*   `view` (可选): `web` 或 `h5` (H5端必填)。

## 4. 授权限制与规则 (重点)
1.  **店铺状态**：授权的店铺必须已完成开店且未发起退店。
2.  **重复授权**：允许，但会导致历史获取的 `code`、`access_token` 及 `refresh_token` **立即失效**。
3.  **取消授权**：商家可以在“拼多多服务市场-授权管理”中取消。
4.  **授权时长与数量限制** (以“三方工具”/“二方系统”为例)：
    *   **开发/审核阶段**：`access_token` 有效期仅 **24小时**，最多只允许授权 **5个** 账号。
    *   **已上线阶段**：三方工具为24小时（可授权25个），二方系统为 **1年**（仅可授权5个）。
    *   **发布到服务市场后**：三方工具的有效期与商家订购时长一致，授权数量不限。
5.  **刷新 Token**：使用 `pdd.pop.auth.token.refresh` 接口刷新，但**不会延长** `access_token` 的绝对有效期限（总有效期与初次授权时长一致）。

## 5. 应用创建准备材料
从截图可知，在 PDD 开放平台创建“商家后台系统”应用，需要准备：
1.  **基础信息**：应用名称（不能涉及拼多多官方）、应用说明。
2.  **设计素材**：应用图标 (260*260，5MB以内，jpg/png)。
3.  **技术准备**：回调地址 (支持 http/https)。
4.  **审核材料 (重点/难点)**：
    *   **PRD 文档** (产品需求文档，pdf格式，<5MB)。
    *   **软件视频** (<100MB的视频文件)，包括核心功能录屏（需包含商品/订单模块），并且需要露出软件名称等。
